Aplicação web - Aprendendo sobre go

Go não é uma linguagem orientada a objetos (OO) como C ++, Java, Ruby e C #. Não possui objetos nem herança e portanto, não tem muitos conceitos associados a OO, como polimorfismo e sobrecarga. O que Go tem são estruturas, que podem ser associadas a métodos. Go também oferece suporte a um formulário simples, mas eficaz de composição. No geral, isso resulta em um código mais simples, mas haverá ocasiões em que você perderá um pouco do que OO tem que oferta. (Vale a pena ressaltar que composição sobre herança é um antigo grito de guerra e Go é a primeira linguagem que eu tenho usado que assume uma posição firme sobre o assunto.)

Embora Go não faça OO como você está acostumado, você notará muitas semelhanças entre a definição de uma estrutura e de uma classe.

Facilitando o entendimento

Se você é novo no mundo da programação simultânea, tudo pode parecer bastante opressor. Isso exige categoricamente consideravelmente mais atenção e cuidado. O objetivo do Go é tornar isso mais fácil. Goroutines efetivamente abstraem o que é necessário para executar o código simultâneo. Os canais ajudam a eliminar alguns bugs graves que pode acontecer quando os dados são compartilhados, eliminando o compartilhamento de dados. Isso não apenas elimina bugs, mas muda como alguém aborda a programação simultânea. Você começa a pensar sobre a simultaneidade no que diz respeito à passagem de mensagens, em vez de áreas perigosas do código. Dito isso, eu ainda faço uso extensivo dos vários primitivos de sincronização encontrados no sync e sync / pacotes atômicos. Eu acho que é importante estar confortável com ambos. Eu encorajo você a se concentrar primeiro nos canais, mas quando você vir um exemplo simples que precisa de um bloqueio de curta duração, considere usar um mutex ou um mutex readwrite.
